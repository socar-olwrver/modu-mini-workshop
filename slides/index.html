소개 · NestJS 개요
요청 하나가 들어오면 이렇게 흐릅니다

Controller · Service · Repository — 한 묶음이 Module.

HTTP 요청
↓
Controller   HTTP만 안다 (라우팅, 입출력)
↓
Service   비즈 로직 — 결제 가능한가, 잔액 충분한가...
↓
Repository   데이터 — DB든 메모리든 외부 API든
↓
응답
Module
위 셋을 묶는 단위
ParkinglotModule ├── ParkinglotController ├── ParkinglotService └── ParkinglotRepository

@Injectable() export class PaymentService { constructor( private repo: PaymentRepository, private session: ParkingSessionService, ) {} }

타입만 적는다.
어떻게 만들어 넣는지는 컨테이너의 몫.

1. Singleton 기본

2. 같은 클래스 = 같은 인스턴스
두 군데서 받아써도 같은 객체
3. 순환 의존성 → 부팅 시 에러
"Circular dependency detected"

@Module({ imports: [OtherModule], // 다른 모듈에서 빌려옴 controllers: [PaymentController], // HTTP 엔드포인트 providers: [PaymentService, // 이 모듈이 만드는 친구들 PaymentRepository], exports: [PaymentService], // 다른 모듈에 빌려줌 }) export class PaymentModule {}
Provider 4가지 형태
providers: [ PaymentService, // ① 클래스 (90%) { provide: 'CONFIG', useValue: { ttl: 3600 } }, // ② 값으로 { provide: Logger, useClass: ConsoleLogger }, // ③ 다른 클래스로 { provide: 'REDIS', useFactory: () => new Redis(...) }, // ④ 팩토리로 ]
@Global()
@Global() @Module({ providers: [LoggerService], exports: [LoggerService], })
한 번만 import하면 어디서든 씀.
